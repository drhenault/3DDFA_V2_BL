Part I

Face Frontalization

Head pose normalization — de-rotating mouth landmarks

3. Face Frontalization — Concept

Objective

Transform mouth point positions as if the speaker were looking straight at the camera, regardless of actual head rotation.

Approach: Direct 3D Geometry

Use stable facial landmarks (eyes, nose, chin) to compute the 3D orientation of the head, then apply the inverse rotation to the mouth points.

Key Insight: Eyes, nose bridge, and chin are bony structures — their relative positions do not change with facial expression (speech). They form an ideal reference frame.

Stable Reference Points Used

pt 36, 39 — right eye corners
pt 42, 45 — left eye corners
pt 27 — nose bridge top
pt 8 — chin

Mouth Points (transformed)

pt 48-59 — outer mouth (12 pts)
pt 60-67 — inner mouth (8 pts)

A total of 20 mouth points are de-rotated, preserving expression (open/closed/asymmetric) while removing head pose distortion.

4. Step 1 — Computing the Face Coordinate Frame

Three Face Axes

We compute a local coordinate frame of the face from reference landmarks:

X-axis — horizontal direction

x̂ = normalize( eye_right − eye_left )

Vector from left eye center to right eye center.

Z-axis — face normal

ẑ = normalize( x̂ × y_approx )

Cross product gives a vector perpendicular to the face plane (pointing toward camera).

Y-axis — vertical (re-orthogonalized)

ŷ = normalize( ẑ × x̂ )

Guarantees a proper orthonormal frame.

# Eye centers (structurally rigid) left_eye = (p[36] + p[39]) / 2.0 right_eye = (p[42] + p[45]) / 2.0 # X-axis: horizontal face direction x_axis = normalize(right_eye - left_eye) # Approximate Y-axis: nose → chin y_approx = p[8] - p[27] # Z-axis: face normal (cross product) z_axis = normalize(cross(x_axis, y_approx)) # Re-orthogonalize Y-axis y_axis = normalize(cross(z_axis, x_axis))
Why re-orthogonalize?
The y_approx vector (nose→chin) is not guaranteed to be perpendicular to x_axis. The cross product ẑ × x̂ ensures that ŷ is orthogonal to both other axes, forming a valid orthonormal basis.

5. Step 2 — Rotation Matrix & Inverse Transform

Building the Rotation Matrix

The columns of Rface are the face axes expressed in the image coordinate system:

Rface = [ x̂ | ŷ | ẑ ]

This matrix transforms from the canonical (frontal) frame to the current face orientation.

Inverse Rotation

Since R is an orthonormal (orthogonal) matrix, its inverse equals its transpose:

Rinv = RfaceT = Rface−1

Rinv transforms from the current face frame back to the frontal frame.

Orthogonal Matrix Properties

  • R · RT = I (identity matrix)
  • det(R) = 1 (preserves orientation)
  • R-1 = RT (inverse = transpose)
  • Preserves distances and angles between points
# Rotation matrix: columns = face axes R_face = np.column_stack([ x_axis, # column 0 y_axis, # column 1 z_axis # column 2 ]) # Inverse = transpose (R is orthogonal) R_inv = R_face.T
Key: Transposing a 3×3 matrix is an O(1) operation — no expensive matrix inversion needed.

6. Step 3 — De-rotating the Mouth Points

Procedure

  1. Extract 3D mouth points (48-67) from CSV data
  2. Center — subtract centroid (center of mass)
    Pcentered = Pmouth − centroid
  3. Apply inverse rotation
    Pfrontal = Rinv · PcenteredT
  4. Restore position — add centroid back
    Presult = Pfrontal + centroid
Why centering? Rotation must be performed around the local center (mouth centroid), not around the image origin. Without centering, points would drift far from the face.
# Extract 20 mouth points in 3D mouth_3d = np.array([ points_3d[i] for i in range(48, 68) ]) # Centroid (center of mass) centroid = mouth_3d.mean(axis=0) # Center, de-rotate, restore mouth_centered = mouth_3d - centroid mouth_frontal = (R_inv @ mouth_centered.T).T result = mouth_frontal + centroid

7. Mouth Aspect Ratio (MAR)

Definition

MAR measures the degree of mouth opening — analogous to EAR (Eye Aspect Ratio) used for blink detection.

Uses inner mouth landmarks (60-67):

MAR = (|p61−p67| + |p62−p66| + |p63−p65|)
―――――――――――――――――
2 · |p60 − p64|
  • Numerator: 3 vertical distances (top↔bottom) — measure opening
  • Denominator: mouth width (corner↔corner) — normalizes for size
  • MAR ≈ 0: mouth closed
  • MAR > 0: mouth open (higher = wider)

Inner Mouth Landmarks (60-67)

# 3 vertical distances d1 = norm(p[61] - p[67]) d2 = norm(p[62] - p[66]) d3 = norm(p[63] - p[65]) # Mouth width (normalization) d_h = norm(p[60] - p[64]) MAR = (d1 + d2 + d3) / (2 * d_h)

8. Why Calculate MAR from Frontalized Points?

Problem with Raw MAR

MAR calculated from original (raw) points contains noise from head movement:

  • Head turns sideways → mouth appears narrower → false MAR decrease
  • Head tilts down → mouth appears more open → false MAR increase
  • Head nodding → MAR oscillations mimic speech

Solution: MAR after Frontalization

MAR calculated from frontalized points:

  • Head movements → no effect (removed by de-rotation)
  • Every MAR change → actual facial muscle movement
  • Clean signal = better speech detection

9. V-VAD Detection Algorithm (ZCR-Enhanced)

Sliding Window Analysis

We analyze the MAR history over the last N frames (window: 0.3 seconds, ~9 frames):

window = [MARt-N, MARt-N+1, ..., MARt]

Three Detection Conditions

1. MAR Variance > threshold

Var(window) > 0.002

Speaking produces rhythmic mouth open/close → high variance.

2. Mean MAR > threshold

Mean(window) > 0.15

Eliminates false positives from micro-movements of a closed mouth.

3. Zero-Crossing Rate (ZCR) ≥ threshold

ZCR( d/dt MAR ) ≥ 3

Detects rhythmic oscillations characteristic of speech (open-close-open-close).

P(speech) = σ(Var) × σ(Mean) × σ(ZCR)
binary: is_speaking = hysteresis( P(speech) )
class VisualVADDetector: def __init__(self, ...): self.window_size = int(0.3 * fps) # ~9 frames self.var_threshold = 0.002 self.mar_activity_threshold = 0.15 self.hold_frames = int(1.0 * fps) # 30 frames self.min_zcr = 3 def update(self, face_idx, mar): self._history[face_idx].append(mar) values = np.array(history) mar_var = np.var(values) mar_mean = np.mean(values) # ZCR of MAR derivative d = np.diff(values) signs = np.sign(d) zcr = np.sum(np.abs(np.diff(signs)) > 0) raw_speaking = ( mar_var > self.var_threshold and mar_mean > self.mar_activity_threshold and zcr >= self.min_zcr )

10. Zero-Crossing Rate (ZCR) — The Key Discriminator

What is ZCR?

ZCR counts how many times the derivative of MAR changes sign (direction) within the sliding window:

d(MAR) = [MARt+1 − MARt, MARt+2 − MARt+1, ...]
ZCR = count( sign(di) ≠ sign(di-1) )

Why ZCR Helps

Speech produces rhythmic oscillations — the mouth repeatedly opens and closes for each syllable. This creates many direction changes in the MAR signal.

  • High ZCR (≥3): Oscillatory motion → likely speech
  • Low ZCR (0-2): Monotonic motion (yawn, slow open/close) or static → not speech
Key Insight: Variance alone cannot distinguish speech from a single slow mouth opening (e.g., a yawn). ZCR detects the rhythmic pattern that is unique to speech.

11. Hysteresis (Hold Time) — Stabilizing Detection

Problem: Flickering

Without hysteresis, the detector rapidly switches between states during speech:

  • Between syllables: mouth closes for ~50-150ms
  • Plosive consonants (p, b, t, d, k) = momentary closure
  • MAR variance drops briefly → false switch to "inactive"

Solution: Hold Timer

Mechanism analogous to hangover in audio VAD systems:

  1. When raw_speaking = True → reset timer to full value (e.g. 30 frames = 1s)
  2. When raw_speaking = False → decrement timer by 1
  3. State remains ACTIVE as long as timer > 0
# Hysteresis logic if raw_speaking: # Speech detected → reset timer self._hold_counter[face_idx] = \ self.hold_frames # e.g. 30 else: # No speech → count down if self._hold_counter[face_idx] > 0: self._hold_counter[face_idx] -= 1 # Active until timer expires return self._hold_counter[face_idx] > 0

12. Speech Probability — Continuous Confidence Score

From Binary to Continuous

Instead of only a binary active/inactive decision, V-VAD now also outputs a continuous probability P(speech) ∈ [0, 1].

Sigmoid Normalization

Each feature is normalized relative to its threshold using a sigmoid function:

σ(x, c, k) = 1 / (1 + e−k·(x/c − 1))

Properties:

  • At the threshold (x = c): σ = 0.5
  • Well above threshold: σ → 1.0
  • Well below threshold: σ → 0.0
  • Steepness k = 4 gives a smooth but decisive transition

Combined Probability

P(speech) = σ(Var/thr) × σ(MAR/thr) × σ(ZCR/thr)

Multiplicative combination — all features must contribute for high probability. If any single feature is low, the probability drops.

# Sigmoid normalization (k=4) def _sigmoid(x, center, k=4.0): z = k * (x - center) return 1.0 / (1.0 + np.exp(-z)) # Per-feature scores (0.5 at threshold) s_var = sigmoid(mar_var / var_thr, 1.0) s_mar = sigmoid(mar_mean / mar_thr, 1.0) s_zcr = sigmoid(zcr / min_zcr, 1.0) # Combined probability prob = s_var * s_mar * s_zcr

Smoothed Probability

A smoothed version with exponential decay mirrors the hold-time behavior in the continuous domain:

Psmooth(t) = max( Praw(t), Psmooth(t−1) · decay )
# Decay rate matches hold time decay = 1.0 - (1.0 / hold_frames) # Per-frame update smoothed = max(prob, prev_smoothed * decay)
Use case: The probability curve enables threshold-free analysis on the dashboard — you can visually compare P(speech) against Audio VAD and choose optimal thresholds without re-running the algorithm.

15. Parameter Tuning & Performance Analysis

Tuning Methodology

A grid search was performed across 12 recordings (6 speakers) with Audio VAD as ground truth. Key decisions:

  • Window 0.3s: Shorter window → faster reaction, sufficient frames for ZCR
  • var_threshold 0.002: Raised from 0.0005 to filter low-amplitude noise
  • mar_threshold 0.15: Raised from 0.12 to reject false detections from barely-open mouth
  • ZCR ≥ 3: New condition — rejects monotonic movements (yawns, slow head turns)
  • Hold 1.0s: Bridges natural inter-syllable pauses (200-400ms) during continuous speech
Optimization target: Maximize F1 score (harmonic mean of Precision and Recall) to balance FP and FN.

16. Known Limitations & Edge Cases

Challenging Scenarios

ScenarioIssueImpact
Very subtle speakers MAR barely exceeds 0.15 during speech Higher FN — some speech missed
Overlapping speech+silence MAR Some speakers have similar MAR/variance during speech and silence Poor discrimination
Head movement noise Rapid head motion creates variance spikes False positives despite ZCR filter
Lip-sync to music Rhythmic mouth motion without actual voice Cannot distinguish — V-VAD will fire
Fundamental limitation: V-VAD is based on visual cues only. It cannot detect whispering (minimal mouth movement) or distinguish lip-syncing from real speech. It should be used as a complement to audio-based VAD, not a replacement.

17. Summary

Face Frontalization

1. Compute face axes from stable landmarks (eyes, nose, chin)
2. Build rotation matrix R = [x̂ | ŷ | ẑ]
3. Apply RT to centered mouth points
4. Restore centroid position → frontalized mouth

Key Features

  • No external canonical 3D model needed
  • Uses data already available from 3DDFA
  • Preserves expression, removes rotation
  • Computational complexity: O(1) per frame

Visual VAD (ZCR-Enhanced + Probability)

1. Compute MAR from frontalized inner mouth pts
2. Sigmoid-normalize Var, Mean MAR, ZCR → P(speech)
3. Smooth probability with exponential decay
4. Hysteresis (hold 1.0s) → binary active/inactive
