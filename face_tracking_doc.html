Face Tracking Stabilization

Technical documentation of the face index (face_idx) stabilization mechanism in the 3DDFA_V2 video processing pipeline

Hungarian Algorithm Exponential Moving Average Dual-Layer Tracking

1. Problem Statement

The FaceBoxes face detector returns a list of detected bounding boxes each frame, sorted by internal criteria (e.g. confidence score, area). When two speakers are at a similar distance from the camera, their order in the detection list can change from frame to frame:

Frame 100: boxes = [Person A, Person B] → face_idx: A=0, B=1 Frame 101: boxes = [Person B, Person A] → face_idx: B=0, A=1 ← SWAP! Frame 102: boxes = [Person A, Person B] → face_idx: A=0, B=1 Frame 103: boxes = [Person B, Person A] → face_idx: B=0, A=1 ← SWAP!

⚠️ Consequences

The CSV output files (mouth_position.csv, face_position.csv) contain mixed-up coordinates — Person A's data ends up in rows labelled face_idx=1 and vice versa. This makes it impossible to correctly track lip movement, face orientation, and speaker identification downstream.

2. Solution Architecture

A dual-layer stabilization approach is employed: once during video processing (pre-processing) and once as a post-processing correction on the CSV files.

┌────────────────────────────────────────────────────────────────────────┐ │ LAYER 1 (Real-Time) │ │ process_video.py │ │ │ │ FaceBoxes ──→ FaceTracker.stabilize() ──→ stable boxes ──→ CSV │ │ (EMA + Hungarian Algorithm) │ └────────────────────────────────────────────────────────────────────────┘ │ write to CSV ▼ ┌────────────────────────────────────────────────────────────────────────┐ │ LAYER 2 (Post-Processing) │ │ reindex_face_ids.py │ │ │ │ Embeddings ──→ cosine similarity tracking ──→ overwrite CSV │ │ or │ │ Landmarks ──→ position-based tracking ──→ overwrite CSV │ │ (EMA + Hungarian Algorithm) │ └────────────────────────────────────────────────────────────────────────┘

Layer 1 — Pre-processing

The FaceTracker class in process_video.py stabilizes the bounding-box order before any data is written to CSV. It operates on bounding-box centroids smoothed with an Exponential Moving Average (EMA).

Layer 2 — Post-processing

The reindex_face_ids.py script corrects face_idx values in the finished CSV files. It prefers face embeddings (cosine similarity), and when those are unavailable it falls back to position-based tracking (centroid of 68 landmarks).

3. The Hungarian Algorithm

The Hungarian Algorithm (also known as the Kuhn-Munkres method) is a combinatorial optimization algorithm that solves the optimal assignment problem in polynomial time O(n³).

The Assignment Problem

We have n face detections in the current frame and m existing tracks from previous frames. For each pair (detection i, track j) we compute a cost — in our case, the Euclidean distance between centroids. We want to find an assignment of detections to tracks that minimizes the total cost:

minimize   Σassignments   cost(detectioni, trackj) Each detection and each track may be assigned at most once.

Cost Matrix

We construct a matrix C of dimensions n × m, where:

C[i, j] = √( (cxi − txj)² + (cyi − tyj)² ) cx, cy = detection centroid;   tx, ty = smoothed track position (EMA)
Example cost matrix (distance in pixels): Track 0 Track 1 (EMA pos) (EMA pos) Detection 0 │ 12.5 │ 187.3 │ Detection 1 │ 203.1 │ 18.7 │ Optimal assignment (minimum total cost = 12.5 + 18.7 = 31.2): Detection 0 → Track 0 ✓ Detection 1 → Track 1 ✓

Padding to a Square Matrix

The linear_sum_assignment function from scipy works with rectangular matrices. When the number of detections ≠ the number of tracks, we pad the matrix to size max(n, m) × max(n, m) with values of max_distance × 2, so that "dummy" assignments carry a prohibitively high cost and are never preferred.

📐 scipy Implementation

We use scipy.optimize.linear_sum_assignment(cost_matrix), which returns the optimal row indices (row_ind) and column indices (col_ind): 

from scipy.optimize import linear_sum_assignment

# Build cost matrix
cost = np.empty((n_det, n_active), dtype=np.float64)
for i, (dcx, dcy) in enumerate(centers):
    for j, (_, t) in enumerate(active):
        cost[i, j] = np.sqrt((dcx - t['cx'])**2 +
                             (dcy - t['cy'])**2)

# Pad to square matrix
dim = max(n_det, n_active)
pad = np.full((dim, dim), max_distance * 2)
pad[:n_det, :n_active] = cost

# Optimal assignment
rows, cols = linear_sum_assignment(pad)

Distance Threshold

Not every assignment is acceptable — if the cost exceeds max_distance (default: 300 px), we consider the detection unmatched to any existing track and a new track is created. This prevents incorrectly linking distant faces.

4. Exponential Moving Average (EMA)

To prevent the track position from reacting to single-frame detection anomalies (noise, minor bounding-box shifts), the centroid of each track is smoothed using an Exponential Moving Average:

EMAt  =  α · xt  +  (1 − α) · EMAt−1 α = ema_alpha (default 0.3)  |  xt = detection centroid at frame t

High α (e.g. 0.7)

  • Faster response to person movement
  • Higher sensitivity to detection noise
  • Recommended for fast-moving subjects

Low α (e.g. 0.1)

  • More stable track position
  • Slower response to actual movement
  • Recommended when subjects are seated or stationary

✅ Default value: α = 0.3

A trade-off between stability and responsiveness. At 25 FPS with α=0.3, the track response to a new position stabilizes after approximately 8–10 frames (~0.3–0.4 s). This is sufficient to filter out single-frame noise while still keeping up with natural head movement.

5. Layer 1 — FaceTracker in process_video.py

The FaceTracker class is instantiated once before the processing loop and called on every frame. Its task is to reorder the bounding-box list so that the position in the list (= face_idx) consistently corresponds to the same person.

Data Flow

FaceBoxes(frame) │ ▼ boxes = [bbox_A, bbox_B] ← order may change! │ ▼ tracker.stabilize(boxes, frame_idx) │ ├─ Compute centroid of each bounding box ├─ Filter out tracks older than drop_after_frames ├─ Build cost matrix (distance: centroid ↔ EMA track position) ├─ Run Hungarian Algorithm (linear_sum_assignment) ├─ Reject assignments with cost > max_distance ├─ Create new tracks for unmatched detections ├─ Update EMA positions for matched tracks │ ▼ boxes = [bbox_A, bbox_B] ← always the same order! │ ▼ TDDFA → dump to CSV

Key Code Fragments

Tracker Initialization

# Spatial face tracker: reorders detected boxes each frame so that
# face_idx stays consistent even when FaceBoxes changes detection order.
tracker = FaceTracker(ema_alpha=0.3, max_distance=300, drop_after_frames=30)

Usage in the Processing Loop

for i, frame in tqdm(enumerate(reader)):
    frame_bgr = frame[..., ::-1]
    boxes = face_boxes(frame_bgr)

    # Stabilize order — the critical step!
    boxes = tracker.stabilize(boxes, i)

    face_count = len(boxes)
    param_lst, roi_box_lst = tddfa(frame_bgr, boxes)
    ...

Track Position Update (EMA)

# After matching a detection to a track:
cx, cy = centers[r]
old = self._tracks[track_orig_idx]
old['cx'] = self.ema_alpha * cx + (1 - self.ema_alpha) * old['cx']
old['cy'] = self.ema_alpha * cy + (1 - self.ema_alpha) * old['cy']
old['last_frame'] = frame_idx

6. Layer 2 — reindex_face_ids.py

The post-processing stage operates on finished CSV files and serves as a second line of defense. It supports two tracking strategies:

Strategy A: Face Embeddings

Uses precomputed 128-dimensional embedding vectors (from a face recognition model). It compares faces across frames using cosine similarity and assigns stable IDs via the Hungarian algorithm on a cost matrix of 1 − similarity.

Strategy B: Position Tracking (fallback)

Activated automatically when embeddings are placeholder (all zeros) or manually via the --use-position-tracking flag. It computes the centroid from the mean position of all 68 facial landmarks and applies the same EMA + Hungarian scheme.

Automatic Placeholder Detection

🔍 Function _are_embeddings_placeholder()

def _are_embeddings_placeholder(embeddings):
    """Detect whether embeddings are all-zero placeholder vectors."""
    if embeddings.size == 0:
        return True
    norms = np.linalg.norm(embeddings, axis=1)
    # If >90% of vectors are near-zero → placeholder data
    return float(np.mean(norms < 1e-8)) > 0.9

When the face recognition model is not integrated, extract_face_embeddings.py writes np.zeros(128) vectors. This function detects that case and automatically switches to position-based tracking.

Position Tracking — Landmark Centroid

Unlike Layer 1 (which uses bounding-box centroids), Layer 2 computes the centroid as the mean X, Y across all 68 facial landmark points. This provides greater precision because landmarks localize the face more accurately than bounding boxes.

Computing Centroids from Landmarks

centroids = (
    df_mouth
    .groupby(['seconds', 'face_idx'])
    .agg(cx=('x', 'mean'), cy=('y', 'mean'))
    .reset_index()
)

Decision Flow

Load embeddings from .npz │ ▼ Are embeddings placeholder? ── YES ──→ Position-based tracking │ (centroid of 68 landmarks + EMA) │ NO ▼ Embedding-based tracking (cosine similarity + Hungarian) │ ▼ Generate assignment_map: (seconds, face_idx) → stable_track_id │ ▼ Overwrite CSV files: ├── mouth_position.csv ├── face_position.csv └── speaker_identification.csv

CSV Reindexing

Function _reindex_df()

def _reindex_df(df, assignment_map):
    """Replace face_idx using assignment_map
    keyed by (rounded seconds, face_idx)."""
    def _new_idx(row):
        key = (round(float(row['seconds']), 6), int(row['face_idx']))
        return assignment_map.get(key, row['face_idx'])

    df = df.copy()
    df['face_idx'] = df.apply(_new_idx, axis=1).astype(int)
    return df

The entire pipeline is orchestrated by stitch_demo.sh. Face stabilization occurs at two points:

8. Configuration Parameters

process_video.py — FaceTracker

Parameter Default Description
ema_alpha 0.3 EMA smoothing factor. Higher → faster response to movement; lower → greater stability.
max_distance 300 px Distance threshold (pixels) for matching a detection to a track. Beyond this → new track.
drop_after_frames 30 Number of frames without a detection before a track is removed from memory.

reindex_face_ids.py — CLI Arguments

Flag Default Description
--dumps_dir dumps Directory containing the CSV dump files.
--embeddings face_embeddings.npz Path to the face embeddings file.
--similarity-threshold 0.3 Minimum cosine similarity for matching (embedding mode).
--drop-after-frames 60 Number of frames after which a track is removed (both modes).
--use-position-tracking false Force position-based tracking instead of embedding-based.
--max-distance 300 px Distance threshold for position-based mode.
--ema-alpha 0.3 EMA smoothing factor for position-based mode.

💡 Tuning Tips

  • Speakers sitting close together — reduce max_distance to ~150–200 px to avoid confusing nearby faces.
  • Speakers walking around — increase max_distance to ~400–500 px and raise ema_alpha to ~0.5 so the tracker can keep up with movement.
  • Frequent detection gaps — increase drop_after_frames so that tracks are not lost during momentary face occlusions.

9. Glossary

TermDefinition
Bounding Box A rectangle (x1, y1, x2, y2) enclosing a detected face.
Centroid The center point of a bounding box, or the averaged position of facial landmarks.
Cosine Similarity A measure of vector similarity: cos(θ) = (A·B) / (||A|| · ||B||). Range [−1, 1].
EMA Exponential Moving Average — a smoothing filter that gives more weight to recent values.
Embedding A 128-dimensional vector describing the identity of a face (from a face recognition model).
face_idx A 0-based face index in a given frame. Goal: make it stable per person across frames.
Hungarian Algorithm An O(n³) algorithm that solves the optimal assignment problem (Kuhn-Munkres).
Landmark A facial characteristic point — 68 points describing the jaw, eyebrows, nose, eyes, and mouth.
linear_sum_assignment The scipy implementation of the Hungarian algorithm in scipy.optimize.
Track A tracking object that stores the historical (EMA-smoothed) position of a single face.
