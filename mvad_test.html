1. Problem — Multi-Voice Activity Detection

The goal of the system is to classify each audio frame (10 ms) into one of three classes:

Class 0
Silence / Noise
No speech activity
Class 1
Single Speaker
One active speaker
Class 2
Overlap (≥2)
Overlapping speech from multiple speakers
Dataset: Synthetically generated WAV files (48 kHz) with precise sample-level ground-truth labels. Split: 300 training files, 50 validation files, 100 test files. Each file is approximately 15 s long and contains randomly mixed segments of silence, single-speaker speech, and overlapping speech from two or more speakers.

2. Method 1 — Traditional (Signal Processing)

SIGNAL PROCESSING

The traditional method is entirely based on hand-crafted acoustic features and heuristic rules. It does not require any machine learning — instead, it leverages domain knowledge about speech signal properties to distinguish silence from single-speaker speech and overlapping speech from multiple speakers.

Key idea: overlapping speech causes characteristic changes in the signal — decreased harmonicity (HNR), wider bandwidth, increased sub-band entropy, F0 instability, and the emergence of a second fundamental frequency. The algorithm combines multiple such indicators into a single overlap score, which is then compared against a threshold.

Processing Pipeline

1
Pre-processing: High-pass filter (80 Hz, 4th-order Butterworth) to remove DC offset and low-frequency noise + pre-emphasis (α = 0.97) to boost higher frequencies.
2
Framing: Audio is divided into 30 ms frames with a 10 ms hop — yielding ~100 frames/s.
3
Vectorized features: STE (Short-Time Energy), ZCR (Zero-Crossing Rate), delta-energy (ΔE) — computed simultaneously for all frames.
4
Silence detection: Adaptive energy threshold (15th percentile + 8 dB) separates speech frames from silence.
5
Per-frame feature extraction — 8 acoustic features (see table below).
6
Adaptive thresholds: Thresholds (25th/75th percentile) are derived from the feature distributions across speech frames of the current file.
7
Overlap scoring: Weighted sum of 8 indicators → score 0…1. Weights: Multi-pitch (0.40), HNR (0.25), F0-var (0.12), Sp.flux (0.12), Bandwidth (0.08), Flatness (0.06), ΔEnergy (0.06), SubBandEntropy (0.06).
8
Temporal smoothing: Max-pool ±3 frames (hangover effect) + moving average (kernel 5).
9
Classification: score ≥ 0.38 → overlap; pitch conf. < 0.25 → soft-recovery / silence; rest → single speaker.
10
Post-processing: Median filter (kernel 7), removal of short overlap segments (< 80 ms) and short speech segments (< 100 ms).

Acoustic Features Used

YIN Pitch Tracking (F0)Fundamental frequency + detection confidence. YIN algorithm with threshold 0.15.
Multi-Pitch DetectionHarmonic sieve on YIN candidates — detects a second voice.
HNR (Harmonic-to-Noise Ratio)Overlap reduces signal harmonicity.
Spectral BandwidthOverlap broadens the frequency spectrum.
Spectral FlatnessMeasure of spectral “noisiness” — overlap makes the spectrum more noise-like.
Sub-Band EntropyEntropy of energy across 6 sub-bands — overlap distributes energy more uniformly.
Delta Energy (ΔE)Rapid energy increase — indicates the onset of a second speaker.
Pitch Variance (F0 var)F0 instability in a ±5-frame window — the tracker struggles during overlap.
Key characteristic: No training phase required — it works immediately on any data. Thresholds adapt per file based on percentiles. The downside is limited generalization capability and manually tuned overlap indicator weights.

3. Method 2 — Deep Neural Network (DNN)

DEEP LEARNING / CNN

The DNN method uses a convolutional neural network (CNN) trained on log mel-filterbank energies (40 mel bands). The network treats a context window (15 frames = 150 ms) as a small “image” of dimensions 15 × 40 and classifies the central frame into one of the 3 classes.

Unlike the traditional method, the network learns by itself which features are important for distinguishing silence, speech, and overlap — no manual rule definition is required. The model is trained with supervision on synthetic data with precise per-sample labels.

Processing Pipeline

1
STFT: Short-Time Fourier Transform (Hann window 25 ms, hop 10 ms, N_FFT = 2048) → power spectrogram.
2
Mel-Filterbank: Triangular bank of 40 mel filters (80–8000 Hz) → logarithmic energy in mel bands → matrix (n_frames × 40).
3
Z-score normalization: Feature standardization based on training set statistics (μ, σ computed offline).
4
Context window: For each frame, ±7 neighboring frames are gathered → patch of 15 × 40 (150 ms of context).
5
CNN forward pass: Patch as a 1-channel image → 3 convolutional blocks → AdaptiveAvgPool → FC → 3-class logits.
6
Argmax: Class with the highest logit → prediction (0/1/2).

Main weakness of the traditional method: Confusing overlap with single speaker (24,496 GT:overlap frames classified as single). DNN drastically reduces this error (down to 2,622 frames). 9. Conclusions

DNN Wins Across All Metrics

The CNN model achieves 94.1% accuracy vs 63.4% — a lead of over 30 percentage points. The improvement is especially impressive on the hardest class — overlap F1 rose from 43.7% to 91.8% (+48.1 pp).

DNN Stability

The standard deviation of per-file accuracy for DNN is only 3.6% (vs 10.8% for the traditional method). The worst-case DNN file reaches 82.8%, while the traditional method drops as low as 31.8%.

DNN Is Faster

Thanks to GPU vectorization and batching, DNN processes 100 files in 29.8 s (RTF = 0.020), while the traditional method requires 119.0 s (RTF = 0.079). That is a 4× speedup.

Advantage of the Traditional Method

The traditional method requires no training — it works out of the box on new data. It is fully interpretable (visible weights and thresholds). In binary detection (speech vs silence) it already achieves good results (F1 = 94.6%).
